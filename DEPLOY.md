# Deployment Guide

## Backend ā†’ Render (Docker)

### 1. Push code to GitHub
Make sure the repo is on GitHub. Render will pull from it.

### 2. Create Render Web Service
- Go to https://render.com ā†’ New ā†’ Web Service
- Connect your GitHub repo
- **Root Directory:** `backend`  (or leave blank and set Docker context below)
- Render auto-detects `render.yaml` if you use **Blueprint** deploy (recommended)

### 3. Blueprint deploy (easiest)
- Render ā†’ New ā†’ Blueprint
- Select your repo ā€” Render reads `render.yaml` automatically

### 4. Set environment variables on Render dashboard
These are marked `sync: false` in render.yaml ā€” you must set them manually:

| Variable | Value |
|---|---|
| `MONGO_URI` | Your MongoDB Atlas connection string |
| `CORS_ORIGINS` | Your Vercel frontend URL, e.g. `https://your-app.vercel.app` |
| `APP_URL` | Same Vercel URL (used in QR code links) |
| `CLOUDINARY_CLOUD_NAME` | From Cloudinary dashboard |
| `CLOUDINARY_API_KEY` | From Cloudinary dashboard |
| `CLOUDINARY_API_SECRET` | From Cloudinary dashboard |

`JWT_ACCESS_SECRET` and `JWT_REFRESH_SECRET` are auto-generated by Render.

### 5. MongoDB ā€” use MongoDB Atlas (free tier)
- Create cluster at https://mongodb.com/atlas
- Whitelist `0.0.0.0/0` (Render IPs change)
- Copy connection string ā†’ set as `MONGO_URI`

### 6. Your backend URL
After deploy: `https://restaurant-os-api.onrender.com`  
Note it down ā€” you need it for the frontend step.

## Frontend ā†’ Vercel (Angular)

### 1. Update environment.prod.ts
Edit `frontend/src/environments/environment.prod.ts`:
```ts
export const environment = {
  production: true,
  apiUrl: 'https://YOUR_RENDER_SERVICE.onrender.com/api',
  socketUrl: 'https://YOUR_RENDER_SERVICE.onrender.com'
};
```
Replace `YOUR_RENDER_SERVICE` with your actual Render service name.

### 2. Import environment in code
Make sure `api.service.ts` and `socket.service.ts` use `environment.apiUrl` / `environment.socketUrl`.

### 3. Deploy to Vercel
- Go to https://vercel.com ā†’ New Project ā†’ Import GitHub repo
- **Root Directory:** `frontend`
- Vercel reads `vercel.json` automatically ā€” no extra config needed
- Build command: `npm run build -- --configuration production`
- Output dir: `dist/ros/browser`

### 4. After deploy
- Copy your Vercel URL (e.g. `https://your-app.vercel.app`)
- Go back to Render and update `CORS_ORIGINS` and `APP_URL` to this URL
- Trigger a Render redeploy

## Socket.IO on Render free tier ā€” important
Render free services sleep after 15 min of inactivity.  
Upgrade to **Starter ($7/mo)** to keep it always-on, which is required for real-time Socket.IO to work reliably.
